One minute starting guide

This guide is for AssertJ core module, you can also find quick start guides for:

  • Guava assertions quick start
  • Joda Time assertions quick start
  • Neo4J assertions quick start

1 - Get AssertJ Core assertions

Maven

AssertJ artifacts are in Maven central repository.

<dependency>
  <groupId>org.assertj</groupId>
  <artifactId>assertj-core</artifactId>
  <!-- use 3.0.0 for Java 8 projects -->
  <version>2.1.0</version>
  <scope>test</scope>
</dependency>

Gradle

For Gradle users (using Maven Central Repository)

testCompile 'org.assertj:assertj-core:2.1.0'

Or version 3.0.0 for Java 8 projects

testCompile 'org.assertj:assertj-core:3.0.0'

If you use another dependency tool, please check this page to find the relevant assertj dependency declaration for your tool.

2 - Add AssertJ methods static import

One static import to rule them all ...

import static org.assertj.core.api.Assertions.*;

... or many if you prefer

import static org.assertj.core.api.Assertions.assertThat;  // main one
import static org.assertj.core.api.Assertions.atIndex; // for List assertions
import static org.assertj.core.api.Assertions.entry;  // for Map assertions
import static org.assertj.core.api.Assertions.tuple; // when extracting several properties at once
import static org.assertj.core.api.Assertions.fail; // use when making exception tests
import static org.assertj.core.api.Assertions.failBecauseExceptionWasNotThrown; // idem
import static org.assertj.core.api.Assertions.filter; // for Iterable/Array assertions
import static org.assertj.core.api.Assertions.offset; // for floating number assertions
import static org.assertj.core.api.Assertions.anyOf; // use with Condition
import static org.assertj.core.api.Assertions.contentOf; // use with File assertions

You can even configure your IDE, so that when you type as and trigger code completion, it will suggest assertThat, see this tip.

3 - Use code completion

Type assertThat followed by the object to test and a dot ... and any Java IDE code completion will show you all the assertions available. 

assertThat(objectToTest). // code completion -> assertions specific to objectToTest

That's all !

4 - Want to see more ?

See some of AssertJ noteworthy features to get the maximum of it !

You can learn more by looking at AssertJ core javadoc.

Another way is to go see assertj-examples tests, it covers what is possible with AssertJ Assertions with running code, you can clone it and run its tests ! (you need to run `mvn install` as it also illustrate how to generate custom assertions.)
